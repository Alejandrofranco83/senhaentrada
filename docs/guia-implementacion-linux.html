3. Instalación del Servidor

Abrir una Terminal en la PC servidor y ejecutar los siguientes bloques, uno por uno:

3a. Actualizar el sistema

# Actualizar paquetes (puede demorar unos minutos)
sudo dnf update -y

3b. Instalar Git y herramientas de compilación

# Git para descargar el proyecto + herramientas para compilar módulos nativos
sudo dnf install -y git gcc-c++ make python3

3c. Instalar Node.js 22 LTS desde NodeSource

# Agregar el repositorio oficial de NodeSource para Node 22
curl -fsSL https://rpm.nodesource.com/setup_22.x | sudo bash -

# Instalar Node.js (incluye npm)
sudo dnf install -y nodejs

# Verificar que quedó bien instalado
node -v
npm -v
Verificación: node -v debe mostrar v22.x.x y npm -v algo como 10.x.x.

3d. Clonar el proyecto e instalar dependencias

# Ir a la carpeta personal
cd ~

# Descargar el proyecto desde GitHub
git clone https://github.com/Alejandrofranco83/senhaentrada.git

# Entrar a la carpeta
cd senhaentrada

# Instalar dependencias
npm install

# Iniciar el servidor (prueba)
npm start
Verificación: Debe aparecer un cuadro con "Sistema de Turnos / Sistema de Senhas" y las URLs de las pantallas. Para detener el servidor de prueba, presionar Ctrl + C.

3e. Obtener la IP del servidor

Abrir otra terminal (o detener el servidor con Ctrl + C) y ejecutar:

ip addr show

Buscar la línea inet 192.168.X.X/24 en la interfaz de red principal (suele llamarse enp..., eth0 o wlan0). Anotar esa dirección IP — se usará en todos los pasos siguientes.

Mi IP del servidor: ________________________________

3f. Abrir el puerto 3000 en el firewall

Fedora viene con firewall activado por defecto. Hay que permitir el puerto 3000:

# Abrir el puerto 3000 de forma permanente
sudo firewall-cmd --permanent --add-port=3000/tcp

# Recargar el firewall para aplicar el cambio
sudo firewall-cmd --reload

# Verificar que el puerto quedó abierto
sudo firewall-cmd --list-ports
Verificación: --list-ports debe incluir 3000/tcp. Sin este paso, las demás PCs de la red no van a poder acceder al servidor.

4. Inicio Automático del Servidor (systemd)

Para que el servidor arranque solo al encender la PC y se reinicie automáticamente si se cae, vamos a crear un servicio systemd.

4a. Averiguar el nombre del usuario actual

whoami

Anotar el resultado (ejemplo: farmacia). Se usa en el siguiente paso.

Mi usuario Linux: ________________________________

4b. Crear el archivo de servicio

sudo nano /etc/systemd/system/senhaentrada.service

Pegar el siguiente contenido (reemplazar TU_USUARIO por el nombre anotado arriba):

[Unit]
Description=Sistema de Turnos Farmacia Franco
After=network.target

[Service]
Type=simple
User=TU_USUARIO
WorkingDirectory=/home/TU_USUARIO/senhaentrada
ExecStart=/usr/bin/node server.js
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target

Guardar y salir: Ctrl + O → Enter → Ctrl + X

4c. Activar y arrancar el servicio

# Recargar systemd para que detecte el nuevo servicio
sudo systemctl daemon-reload

# Habilitar el servicio para que arranque al encender la PC
sudo systemctl enable senhaentrada

# Arrancar el servicio ahora
sudo systemctl start senhaentrada

# Ver el estado (debe decir "active (running)" en verde)
sudo systemctl status senhaentrada
Verificación: Reiniciar la PC con sudo reboot y verificar que el servidor arranca solo. Probar desde otra PC entrando a http://192.168.X.X:3000/admin.

Comandos útiles del servicio

Qué hacerComando

5. Configurar la Impresora

  1. Conectar la impresora al router con cable de red
  2. Encender la impresora — generalmente imprime un ticket con su configuración de red
  3. Anotar la IP de la impresora (ejemplo: 192.168.1.100)
  4. En el PC servidor, editar el archivo de configuración: 
    nano ~/senhaentrada/config/default.json
  5. Cambiar la sección de impresora: 
    {
  "printer": {
    "enabled": true,
    "type": "network",
    "ip": "192.168.1.100",
    "port": 9100
  }
}
  6. Guardar (Ctrl + O → Enter → Ctrl + X) y reiniciar el servicio: 
    sudo systemctl restart senhaentrada

6. Configurar Operadores y Cajas (Panel Admin)

Desde cualquier PC del local, abrir el navegador e ir a:

http://192.168.X.X:3000/admin

  1. Ir a la pestaña "Operadores"
  2. Editar cada operador con su nombre real
  3. Subir la foto de cada operador (botón "📷 Foto")

7. Configurar la Smart TV (Display)

  1. Encender la TV y conectarla al WiFi del local
  2. Abrir el navegador de la Smart TV
  3. Ir a: http://192.168.X.X:3000/display
  4. Tocar "Activar Pantalla / Ativar Tela" (habilita el audio)
  5. Poner en pantalla completa
  6. Subir el volumen de la TV al nivel deseado para los anuncios por voz
Importante: Si la TV se apaga o el navegador se cierra, hay que repetir los pasos 2-5. Se recomienda desactivar el apagado automático de la TV.

8. Configurar la Tablet (Kiosko)

  1. Conectar la tablet al WiFi del local
  2. Instalar "Fully Kiosk Browser" desde Play Store
  3. Abrir Fully Kiosk y configurar:
    • Start URL: http://192.168.X.X:3000/kiosk
    • Enable Kiosk Mode: Activar
    • Kiosk Exit Password: Definir una contraseña (para poder salir del modo kiosko)
  4. Montar la tablet con el soporte en la entrada, junto a la impresora
  5. Conectar el cargador de la tablet (va a estar encendida todo el día)
Contraseña de Fully Kiosk: ________________________________

9. Configurar las PCs de las Cajas (Operador)

En cada PC de caja:

  1. Abrir Chrome, Edge o Firefox
  2. Ir a: http://192.168.X.X:3000/operator
  3. Opcional — crear acceso directo en el Escritorio:
    • Click derecho en Escritorio → Nuevo → Acceso directo
    • Ubicación: chrome "http://192.168.X.X:3000/operator"
    • Nombre: "Sistema de Turnos"
El operador solo necesita: Abrir el acceso directo, seleccionar su nombre, seleccionar su caja, y click "Abrir Caja". Si recarga la página, la sesión se mantiene.

10. Prueba Completa

Seguir estos pasos en orden y marcar cada uno:

  • Un operador abre su caja desde el panel operador
  • Tocar "Farmacia General" en la tablet del kiosko
  • Verificar que se imprime el ticket
  • El operador hace click en "Llamar Siguiente"
  • Verificar que la TV muestra el turno con borde parpadeante
  • Verificar que suena el anuncio de voz (español + portugués)
  • El operador hace click en "Completar"
  • Repetir con "Telefonía y Servicios"
  • Repetir con "Retirada de Mercadería"
  • Probar "Elegir Atendiente" → seleccionar un operador → verificar que solo ese operador recibe el ticket
  • Probar "Pausar" en una caja → verificar que no recibe tickets
  • Verificar estadísticas en Admin

13. Solución de Problemas

ProblemaSolución
No se puede acceder a las pantallas desde otras PCsVerificar que el puerto 3000 está abierto en el firewall (paso 3f). Verificar que todos los dispositivos están en la misma red WiFi/LAN. Probar ping 192.168.X.X desde otra PC.
El servicio no arrancaVer logs: sudo journalctl -u senhaentrada -n 50. Verificar que la ruta en WorkingDirectory del archivo de servicio es correcta.
No imprime el ticketVerificar la IP de la impresora en ~/senhaentrada/config/default.json. Verificar que "enabled" es true. Verificar cable de red. Reiniciar con sudo systemctl restart senhaentrada.
No suena el audio en la TVHay que tocar "Activar Pantalla" cada vez que se recarga la página. Verificar volumen de la TV.
Operador no puede entrar a una cajaIr a Admin → Cajas → "Forzar Cierre" en la caja bloqueada.
Ver qué está haciendo el servidorsudo journalctl -u senhaentrada -f (Ctrl+C para salir)
